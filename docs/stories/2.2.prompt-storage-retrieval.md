# Story 2.2: Prompt Storage & Retrieval

## Status
Draft

## Story
**As a** Product Owner,
**I want** my custom prompts to be securely stored and reliably retrieved by the extension,
**so that** my prompt customizations persist across browser sessions and are immediately available when generating feedback.

## Acceptance Criteria
1. Implement secure storage of custom prompts using existing extension storage API
2. Integrate prompt storage with current settings system maintaining backward compatibility
3. Provide efficient retrieval of prompts based on selected AI provider
4. Include data validation and error handling for prompt storage operations
5. Support import/export functionality for prompt templates across different browser installations

## Tasks / Subtasks
- [ ] Extend existing storage schema for custom prompts (AC: 1, 2)
  - [ ] Define storage structure for provider-specific custom prompts
  - [ ] Implement migration logic for existing settings without custom prompts
  - [ ] Add default prompt templates as fallback values
  - [ ] Ensure storage encryption/security follows browser extension best practices
- [ ] Enhance OptionsManager with prompt storage methods (AC: 1, 3)
  - [ ] Add saveCustomPrompt(provider, prompt) method to OptionsManager
  - [ ] Add getCustomPrompt(provider) method with fallback to defaults
  - [ ] Implement validatePrompt(prompt) for template syntax validation
  - [ ] Add bulk operations for saving multiple provider prompts
- [ ] Implement retrieval integration with background script (AC: 3)
  - [ ] Modify background.js sendToLLM function to use custom prompts
  - [ ] Add prompt selection logic based on current provider setting
  - [ ] Implement caching mechanism for frequently used prompts
  - [ ] Add fallback to default prompts when custom prompts fail validation
- [ ] Add comprehensive data validation (AC: 4)
  - [ ] Validate prompt template syntax and required variables
  - [ ] Implement error handling for storage quota exceeded scenarios
  - [ ] Add validation for template variable substitution
  - [ ] Create user-friendly error messages for validation failures
- [ ] Implement import/export functionality (AC: 5)
  - [ ] Add export functionality to save all custom prompts as JSON file
  - [ ] Implement import functionality to load prompts from JSON file
  - [ ] Add validation for imported prompt files
  - [ ] Create backup/restore functionality for prompt configurations
- [ ] Update existing integration points (AC: 2)
  - [ ] Ensure feedback.js can access and use custom prompts
  - [ ] Update options page to load/display existing custom prompts
  - [ ] Maintain compatibility with existing API provider switching
  - [ ] Test storage operations with existing chrome.storage.local usage
- [ ] Add comprehensive testing (AC: 1-5)
  - [ ] Unit tests for storage schema migration
  - [ ] Unit tests for prompt CRUD operations

### Existing Storage Implementation
[Source: docs/stories/1.2.provide-feedback.md#dev-notes]
Current storage usage patterns:
- chrome.storage.local.set() for saving settings
- chrome.storage.local.get() for loading settings  
- Auto-save on input changes in OptionsManager
- Storage schema: `{ apiProvider, apiKey, customEndpoint }`

### Storage Schema Design
[Source: Analysis of current implementation]
Extended storage schema:
```javascript
{
  // Existing fields (maintain backward compatibility)
  apiProvider: 'openai|anthropic|custom',
  apiKey: 'string',
  customEndpoint: 'string',
  
  // New fields for custom prompts
  customPrompts: {
    openai: 'template string with {{variables}}',
    anthropic: 'template string with {{variables}}',
    custom: 'template string with {{variables}}'
  },
  promptVersion: '1.0', // For future migrations
  promptBackups: [] // For backup/restore functionality
}
```

### Default Prompt Templates
[Source: Background script analysis]
Each provider needs default templates with standard variables:
- `{{storyContent}}` - The extracted user story content
- `{{userRequirements}}` - Any additional user requirements
- `{{feedbackType}}` - Type of feedback requested (critique, suggestions, etc.)

### Import/Export File Format
JSON structure for prompt import/export:
```javascript
{
  version: '1.0',
  exportDate: '2025-08-01T00:00:00Z',
  prompts: {

    anthropic: 'template string', 
    custom: 'template string'
  },
  metadata: {
    extensionVersion: '1.0.0',
    userNotes: 'optional user description'
  }
}
```
